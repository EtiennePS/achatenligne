Les requêtes suivantes permettent de tester un scénario nominal pour l'API :
- Création d'un produit de test
- Récupération des info du produit de test
- Récupération des infos de tout les produits
- Modification du produit
- Suppression du produit

Si la base de donnée ne vient pas d'être (re)créé, l'id ne sera pas 1 et devra être modifié. Une variable a été créé pour simplifier les modifications.

idProduit=1
curl -v --request POST --data-binary '{"code":"TEST","libelle":"produit de test","prix":100}' --header "Content-Type: application/json" http://localhost:8080/achatenligne/api/produit
curl -v --request GET http://localhost:8080/achatenligne/api/produit/${idProduit}
curl -v --request GET http://localhost:8080/achatenligne/api/produit
curl -v --request PUT --data-binary '{"code":"TEST2","libelle":"produit de test modifié","prix":111}' --header "Content-Type: application/json" http://localhost:8080/achatenligne/api/produit/${idProduit}
curl -v --request DELETE http://localhost:8080/achatenligne/api/produit/${idProduit}
